# PatternDefinition

The `PatternDefinition` record encapsulates the metadata, classification, and detection results for a security rule within the `skill-guard` scanning engine. It serves as the primary contract for defining how a specific security pattern is identified, categorized, and how the results of that pattern's execution are structured.

## API

### Id
`public abstract string Id`
The unique identifier for the pattern definition.

### Name
`public abstract string Name`
The human-readable display name of the security pattern.

### Description
`public abstract string Description`
A detailed summary explaining the purpose of the security pattern and the types of issues it is designed to detect.

### DefaultSeverity
`public abstract Severity DefaultSeverity`
The default `Severity` level associated with any findings generated by this pattern, unless overridden by configuration.

### Category
`public abstract FindingCategory Category`
The `FindingCategory` that classifies the type of security concern this pattern addresses.

### Scan
`public IEnumerable<Finding> Scan`
The collection of `Finding` objects generated by the pattern, representing the results of the analysis.

## Usage

### Accessing Pattern Metadata

```csharp
public void LogPatternInformation(PatternDefinition pattern)
{
    Console.WriteLine($"Rule ID: {pattern.Id}");
    Console.WriteLine($"Name: {pattern.Name}");
    Console.WriteLine($"Severity: {pattern.DefaultSeverity}");
    Console.WriteLine($"Category: {pattern.Category}");
}
```

### Iterating Over Scan Results

```csharp
public void ProcessFindings(PatternDefinition pattern)
{
    foreach (var finding in pattern.Scan)
    {
        Console.WriteLine($"[ {pattern.Name} ] Detected issue at: {finding.Location}");
        Console.WriteLine($"Description: {finding.Message}");
    }
}
```

## Notes

- **Immutability:** As `PatternDefinition` is a `sealed record`, its properties are immutable after instantiation, ensuring thread safety for read operations.
- **Data Integrity:** The `Scan` collection should be treated as read-only. Implementations returning `IEnumerable<Finding>` should ensure that the results are consistent for the lifetime of the `PatternDefinition` instance.
- **Evaluation Cost:** Depending on the implementation, the `Scan` property may involve deferred execution. Iterating over the collection multiple times may trigger re-evaluation of the detection logic if the underlying implementation is not cached.
